# 測試流程

這份文件描述 lottery-capybara 的最小可重複驗證流程。因為專案是純靜態前端，所以測試重點放在：

- JavaScript 語法是否有效
- 頁面是否能正常渲染
- 抽獎流程是否正確
- 重置與再次抽獎是否符合預期

## 測試前提

- 已安裝現代瀏覽器
- 可直接開啟 [index.html](index.html)
- 若要用命令列做語法檢查，需要安裝 Node.js

## 快速檢查

在專案目錄執行：

```bash
node --check script.js
```

預期結果：

- 指令沒有輸出錯誤
- 結束碼為 0

## 手動測試流程

### 1. 初始載入

操作：

1. 開啟 [index.html](index.html)

預期：

1. 頁面標題顯示「卡比巴拉抽獎大會」
2. 畫面上有 8 張角色卡
3. 抽出人數初始值為 1
4. 參與人數顯示為 8 / 8
5. 減號按鈕為 disabled

### 2. 切換參與資格

操作：

1. 點擊任一張角色卡

預期：

1. 該卡片變成 inactive 樣式
2. 參與人數減 1
3. 若目前抽出人數大於剩餘參與人數，抽出人數會自動收斂到可用範圍

### 3. 調整抽出人數

操作：

1. 點擊加號
2. 點擊減號

預期：

1. 數字會正確增減
2. 不可小於 1
3. 不可大於目前參與人數

### 4. 單次抽獎

操作：

1. 保持至少 1 位參與者
2. 點擊「開始抽獎！」

預期：

1. 開始抽獎後按鈕會暫時 disabled
2. 卡片會有掃描高亮動畫
3. 抽獎完成後會出現得獎彈窗
4. 得獎卡片上會出現獎盃標記

### 5. 多人抽獎

操作：

1. 把抽出人數調成 2 或以上
2. 點擊「開始抽獎！」

預期：

1. 得獎彈窗中會列出多位得獎者
2. 主畫面上對應數量的卡片會被標記為得獎
3. 不會抽到重複的人

### 6. 再抽一次

操作：

1. 在得獎彈窗按「再抽一次」

預期：

1. 彈窗關閉
2. 上一輪得獎標記清除
3. 目前被排除的參與者狀態應保留
4. 抽出人數應保留

### 7. 重置

操作：

1. 點擊「重置」

預期：

1. 所有角色恢復參與
2. 參與人數顯示回到 8 / 8
3. 抽出人數回到 1
4. 所有得獎標記與高亮狀態被清除
5. 得獎彈窗關閉

## 回歸測試重點

每次修改後，至少重跑以下項目：

1. `node --check script.js`
2. 初始載入
3. 排除 1 人後抽 2 人
4. 關閉得獎彈窗
5. 重置回初始狀態

## 常見故障判斷

### 頁面有內容但卡片沒出現

優先檢查：

1. [script.js](script.js) 是否載入失敗
2. 瀏覽器 console 是否有語法錯誤
3. [index.html](index.html) 的 `<script src="script.js">` 是否正確

### 頁面排版完全跑掉

優先檢查：

1. [style.css](style.css) 是否載入成功
2. [index.html](index.html) 的 `<link rel="stylesheet" href="style.css">` 是否正確

### 抽獎結果異常

優先檢查：

1. 參與人數與抽出人數是否同步
2. 洗牌與 slice 邏輯是否被改壞
3. 重置是否殘留上一輪狀態
